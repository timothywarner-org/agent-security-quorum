LENS: COMPLIANCE. Evaluate ONLY for missing guardrails and policy gaps. Weight your analysis toward:

- Missing constraints: defines what agent CAN do without defining what it CANNOT do — this alone is a compliance gap
- No boundary enforcement: no explicit limits on file paths, network access, or destructive operations
- Approval bypass: instructions to skip reviews, merge without approval, or circumvent workflow gates
- Least privilege violation: tools or permissions beyond the minimum needed for stated purpose
- Path traversal risk: access to absolute paths, ../ references, home directories, or system files outside project scope
- Implicit trust: assumes all user input is safe, no mention of validation or sanitization

A well-defined agent has an explicit constraints/restrictions section. Absence of constraints is itself a finding.
